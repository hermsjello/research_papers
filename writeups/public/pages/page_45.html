Hello Everyone,

I’m MRD7

Do you know there are some classes of bug which are so easy to find that you don’t even need any proxy tool to find them. All you need to pay attention in URLS.

Anyone can find this type of bug using mobile phone also.

Today, I’m gonna share with you guys one of a very simple bug. This is a bug, which is too easy but mostly ignored i.e. Forced Browsing

What is Forced Browsing?

Forced browsing is an attack where the aim is to enumerate and access resources that are not referenced by the application, but are still accessible.

You can read more about it here:

Forced browsing

Let’s talk about how did I exploit it?

So, I got an invite to a private program where admin could invite members and admin’s team would remain private.

For example, if admin team id = 1234

And if you try to visit it directly like, it throws error

https://www.redacted.com/team/1234 → Forbidden

So, as an attacker my goal was to find a way to identify private team members of other [victim] admin’s team.

At first, I tried to look for IDOR in some parameter in request that could leak other admin’s team member. But didn’t find any IDOR through parameter in request.

Now, from attacker account, I started browsing my own [attacker’s] team links and started noticing all URLs and one URL stood out.

Vulnerable URL:

https://www.redacted.com/teams/1337/d751713988987e9331980363e24189ce?page=1&per_page=10

Breakdown of vulnerable URL:

https://www.redacted.com/teams/<4-digit-team_id>/<unique-long-key>?page=1&per_page=10

<4-digit-team_id> → This was the 4 digit team id. So, it was easy to bruteforce.

<unique-long-key> → This was the long unique key which was like authentication token used by the application in backend to validate the team.

This unique-long-key had no validation. And because of that, attacker could use this token to leak other admin’s private team member.

POC:

  1. From attacker account visit https://www.redacted.com/teams
  2. The above URL will redirect to https://www.redacted.com/teams/<team-id>/<unique-long-key>/
  3. Copy the long <unique-long-key>
  4. Now enter that vicitm admin’s team id [team id was just 4 digit long] Ex 1234
  5. https://www.redacted.com/teams/1234/d751713988987e9331980363e24189ce?page=1&per_page=10

6. In response, all members of victim admin’s private team will be disclosed.

Lesson: “ You don’t always need proxy tools and big setup to find bugs. Just notice everything. ”
