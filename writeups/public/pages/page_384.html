The idea behind business logic vulnerabilities is that one should be able to abuse the way how an application handles requests and responses. Earlier this year i found out about a vulnerability in a Hackerone’s private program which allowed me to impersonate users in Support requests.

To protect the privacy of the program, let’s call the website vulntome.com. To start off as usual, while testing the website i was looking for peculiar behavior in the application. I found out two things

  1. The support page of the application requires an email address if you are logged in
  2. The support page does not ask for the email address if you are not logged in

Next target was to observe the request within session and without

Request without session

Host: vulntome.com

If you observe both requests, you will notice that there is a different parameter called “userid”. If a user is not logged in or does not have an account, the support request can be sent to any email, and that is not actually a flaw but design. but if a user is logged in and the userid can be changed to that of some other user, then there could be potentially a chance to request support on behalf of any other user.

So plan of attack was this:

I created two accounts

Both had unique user ids

Went to the support page and request support of password reset

Intercepted the request with burp, changed the “userid” to that of the second account

The support could be request on behalf of the second account

How would this affect the application, Below are a couple of scenarios

The attacker requests internal information from the support to another communication channel

The attacker could request a password change from the support page by manipulating the id

The attacker could potentially perform any malicious task on behalf of the other user.

I timely reported the flaw to the program and they acknowledged it, fixing it quickly and rewarding me with $4250

Lessons to be learned from this. The bounty that i got for this vulnerability is equivalent to the bounty rewarded for an RCE in some conditions, this proves that given the right reporting skill and better finding methology, Business logic flaws can also win big bucks
